## 命名决策说明

**① `shipping_date` = 送达日期，不是发货日期**
数据验证：`shipping_date − order_date` 恒等于 `days_for_shipping_real`。说明它是**配送完成（货到）那天**，不是发出那天。译成"发货日期"会误导。

**② `shipping_mode` = 运输等级，不是运输方式**
取值是 Standard / First / Second Class、Same Day——是**服务等级**（客户为速度付费的产品分层），不是海运/空运那种"方式"。

**③ `department` = 商品大类，不是部门**
取值是 Fan Shop / Apparel / Golf / Footwear——是**门店商品分类**，不是组织架构里的部门。

**④ 地理字段用"订单"而非"收货"前缀**
取消/欺诈订单从未送达，用"收货城市"会与事实冲突；"订单城市"描述的是订单上的目的地，更稳。（目前 `收货国家`/`收货州` 尚未统一，建议改为"订单国家"/"订单州省"。）

**⑤ `order_item_profit_ratio` = 行级利润率**
Power BI 里另有一个同名**度量值** `[利润率]`（= 总利润 ÷ 净收入，汇总口径）。为避免 DAX 引用混淆，把**列**改名为"行级利润率"：
- **列**（行级）→ 用来给每单贴标签（深亏/浅亏分档）
- **度量值**（汇总）→ 用来算整体盈利效率
- 口诀：**贴标签用列，算总体用度量值**
