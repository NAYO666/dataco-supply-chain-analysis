# 字段清单（D1 产出）

**数据粒度：订单行（order item）**——一行 = 一个订单里的一个商品项。
证据：180519 行 vs `Order Id` 去重后 65752 个订单 → 平均每单 2.75 个商品行。算"订单数"要对 Order Id 去重，算"销售额"才能直接对行求和。

**判断编号**：①利润 ②履约 ③库存　｜　处理：✅保留 ❌删除 🟡存疑
**数据边界（重要）**：无运费成本字段、无罚金/客户流失字段、无库存快照 → 三个判断的口径需相应近似（见 [三个业务判断.md](三个业务判断.md)）。

## ❌ 删除（22 个，D3 清洗时 drop）
> 数量核对：8 + 6 + 5 + 3 = 22，且 53 − 22 = 31（保留列数）对得平。

**隐私 / 打码 / 垃圾值（8）**
Customer Email（打码X）、Customer Password（打码）、Product Description（全空）、Product Image（图片URL）、Customer Fname（姓名PII）、Customer Lname（姓名PII）、Customer Street（门牌PII）、Order Zipcode（大量缺失，且字段说明文件都没列它）

**客户地理冗余（6）**——分析看"货发到哪"，客户注册地无用，留货物侧 Order City/Country/State
Customer City、Customer Country、Customer State、Latitude、Longitude、Customer Zipcode

**重复列（5，已用 `==` 验证完全相同，各留一删一）**
Benefit per order（=Order Profit Per Order）、Order Item Product Price（=Product Price）、Product Category Id（=Category Id）、Order Customer Id（=Customer Id）、Order Item Cardprod Id（=Product Card Id）

**无用列（3）**
Product Status（常量列，全为0）、Type（支付方式，与三判断无关）、Sales per customer（客户累计销售额，可 groupby 现算）
