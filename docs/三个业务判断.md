# 三个业务判断（项目北极星）

定位：**交付物是三个业务判断，「MySQL → Python → Power BI」管道只是取证手段。**
判断 = 现象定位 + 量化影响 + 建议动作。对外表达先讲判断、再讲管道。

## 判断①｜利润：谁在系统性亏钱
- 决策问题：哪些 品类/SKU × 区域 组合在持续产生负利润？累计亏多少？该限折扣、复核定价还是停售？
- 取证路径：D6 负利润下钻 → D9 品类盈利页（可选）→ D13 报告
- ⚠️ 数据现实：DataCo **没有运费成本字段**，「亏运费」不可直接算。口径改为 `Order Profit Per Order < 0` 定位亏损集中区，重点查折扣率（Order Item Discount Rate）与亏损的关系——大概率是折扣打穿了利润
- 产出句式：「X 品类在 Y 区域持续负利润，累计亏 Z 美元，其中 W% 发生在折扣率 > D% 的订单上，建议对该组合限折扣/复核定价」

## 判断②｜履约：迟发集中在哪、值多少钱
- 决策问题：迟发集中在哪个环节（运输方式×市场×承诺天数）？影响多少金额的订单？改什么能降到多少？
- 取证路径：D6 逐维切分 → D10 帕累托定位 → D11 反事实测算 → D9 迟发专题页
- ⚠️ 数据现实：没有罚金/客户流失字段，「值多少钱」= **受影响订单 GMV**（口径写明）；改善效果是历史数据反事实推演，不是因果实验
- 产出句式：「X% 的迟发集中在 Y 组合，对应 Z 美元订单；将该组合的承诺天数/运输方式调整为 W，历史数据推演迟发率可由 A% 降至 B%，假设与边界见 notebook」

## 判断③｜库存：现行策略比最优多占多少资金
- 决策问题：一刀切高服务水平备货 vs ABC-XYZ 差异化备货，安全库存资金占用差多少？
- 取证路径：D12 九宫格 + z×σ 测算
- ⚠️ 数据现实：这是订单数据，**没有库存快照**——不是实际库存审计，是两种备货策略的所需安全库存对比测算。假设必须逐条显式写出：提前期、各档服务水平（如 AX 98% / CZ 85%）、成本口径（用 Product Price 还是推算成本）
- 产出句式：「对 N 个样本 SKU，统一 98% 服务水平的安全库存占用资金 X，差异化策略占用 Y，节省 Z（-W%），建议对 CZ 类降服务水平或按单采购」

> 三条 ⚠️ 不是项目的弱点——能诚实写出测算边界的分析，比吹「我降低了 X%」的可信十倍。
