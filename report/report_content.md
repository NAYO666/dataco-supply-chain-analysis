# DataCo 供应链分析

> 从 180,519 个订单行中提炼三个可执行判断｜Python + MySQL + Power BI

## 核心指标

- 57.31%｜订单级迟发率
- 12.03%｜整体利润率
- 18.71%｜亏损订单行占比
- 115.2 万｜基准安全库存资金

## 01 盈利｜先拦截深亏，不要先砍品类

- **发现**：利润率低于 -50% 的订单行仅占 8.69%，却贡献 85.3% 的亏损；亏损总额 388 万，接近当前净利润 397 万。
- **行动**：利润率低于 -100% 硬拦截，-50% 至 -100% 触发审批；优先在 Fishing 与 Camping 试点。深亏占比在各品类近似均匀，说明它更像系统性定价或履约规则漏洞。

## 02 履约｜先校准承诺，再投入运输能力

- **发现**：First Class 承诺 1 天、实际恒为 2 天，导致订单级迟发率 100%；Second Class 实际时效与 Standard Class 接近。约18%的运输等级×市场×品类组合覆盖约80%的迟发订单行。
- **行动**：先按历史能力校准承诺，订单级迟发率可由 57.3% 降至 31.6%；再把能力投入集中到迟发量最大的 Standard Class 和仍有能力缺口的 Second Class。

## 03 库存｜资金杠杆在提前期，不在服务水平

- **发现**：在102个具有足够销售历史的可测算SKU中，A类7个SKU贡献77.6%的销售额，但只占安全库存资金约 5%；B 类销售额仅 17.9%，却占资金约 89%。
- **行动**：九宫格差异化服务水平反而使资金增加 4.2%；应优先压缩 B 类补货提前期，提前期减半的测算可释放约 30 万资金，并将 BZ 格改为按单采购。
