## D10（帕累托根因下钻）
- 工作内容：`02_late_delivery_analysis.ipynb`。用窗口函数对「运输等级×市场×品类」639 个组合算迟发累计占比（帕累托）；拆解各等级对迟发的"率"与"量"贡献；算受影响 GMV
- 方法与原理：帕累托曲线读法——X 轴是组合按迟发量降序的排名，Y 轴是累计占比，**曲线前陡后平=问题集中**（若是直线则无重点可抓）；"率"是相对承诺的失约比例，"量"是绝对订单数，两者可以完全背离
- 关键收获：**约18%的运输等级×市场×品类组合覆盖约80%的迟发订单行**；⭐**率与量分离**——First Class 订单级迟发率 100%（最高），但只占迟发订单行总量 26.8%；Standard 订单级迟发率 39.8%（最低），却占迟发订单行总量 41.4%（基数最大）。**只治 First Class 最多覆盖约四分之一迟发订单行**，只看率会把火力打错方向；迟发受影响 GMV 约 **2013 万（占总 GMV 57.2%）**

## D11（情景测算 / 反事实）
- 工作内容：设计承诺天数阶梯做反事实推演——历史实际配送天数不变，只改承诺重新判定迟发；分等级识别"承诺问题"vs"能力问题"；写明假设与局限
- 方法与原理：反事实=只改判定标准不改历史事实（像"及格线从90改60，用原分数重划"）；**承诺定在平均值必然约一半失约**，应定分位数；**"承诺定第几分位"就是服务水平**，与 D12 安全库存的 z 值同一数学内核（面对波动，覆盖多少要付多少代价）；用历史推演的前提是系统稳定——迟发率四年无趋势恰好支撑了这个前提
- 关键收获：⭐**承诺阶梯 57.3% →（改成各等级平均）31.6% →（P80）15.9% →（全覆盖）0%**；⭐**两类病分开治**——First Class 配送时间**零波动**（9602 单全部 2 天）却承诺 1 天，纯属过度承诺，**改数字即归零、零成本**；Standard/Second 实际 2-6 天均匀铺开，是**真实能力不足**，要投入才能改善；⭐**这不是预测是上限参照**——只回答"仅改承诺最多能降多少"，真实效果需小范围试点验证；承诺放宽会损失多少时效敏感客户，数据无流失/弹性字段，需业务方补充后做取舍
